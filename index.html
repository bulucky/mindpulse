02 · Features

为什么需要 MindPulse

  1. 余光感知Peripheral awareness

    AI 工具的状态变化通过颜色与动画节奏直接传达 — 酒红停、绿慢转、紫快闪、橙脉冲 — 不需要切窗口、看终端。

  2. 多工具共存Multi-tool tracking

    同时追踪 Claude Code 和 Codex 等多个 AI 工具，状态机以嵌套计数 + 优先级聚合（Pending > Running > Idle > Stopped），最高优先级状态胜出。

  3. 纯过程化渲染Procedural 32×32 icon

    32×32 图标由数学公式实时生成 — 三缺口金比例旋转圆环，径向高斯分布抗锯齿 — 不依赖任何图片资源。

  4. 零依赖二进制Zero runtime deps

    C++17 编译为单个 mindpulse.exe，仅链接 Windows 系统 DLL（user32 / gdi32 / shell32 / ws2_32），无需安装运行时。

  5. 配置热重载Hot-reload YAML

    修改 config/*.yaml 即可调整事件映射或添加新工具支持，无需重启进程。文件缺失时自动生成默认配置。

  6. 本地 HTTP 服务Localhost API

    在 127.0.0.1:9876 监听 POST 钩子事件（兼容 Claude Code 和 Codex 事件格式），同时提供 GET 诊断端点。

core · libcore.a

无平台依赖的纯逻辑层：呼吸动画曲线、状态机转换、YAML 配置加载、图标过程化渲染。

server · libserver.a

HTTP 服务层：接收 POST /hook/<tool_id>，解析 JSON 负载，通过 ConfigManager 映射事件名后送入 StateMachine。

平台 UI 层：Win32 Shell_NotifyIcon 实现系统托盘图标更新与右键菜单；非 Windows 平台降级为 stdout 日志桩。

04 · Quickstart

快速开始

三步让任务栏图标跟随 AI 工具状态呼吸。

  1. 构建或下载

    git clone https://github.com/bulucky/mindpulse → cmake -B build -G Ninja && cmake --build build
    或从 Releases 下载预编译 mindpulse.exe。

  2. 启动服务

    双击 mindpulse.exe — 任务栏出现暗酒红图标，HTTP 服务在 127.0.0.1:9876 开始监听。

  3. 配置 AI 工具的钩子

    在 Claude Code 中运行 /hooks 添加 HTTP 钩子，或在 .claude/settings.json 中配置 hooks 段指向 http://127.0.0.1:9876/hook/claude。

配置示例

# config/claude.yaml
claude:
  name: "Claude Code"
  events:
    SessionStart:       "SESSION_START"
    PreToolUse:         "TOOL_START"
    PostToolUse:        "TOOL_END"
    PermissionRequest:  "PERMISSION_REQUEST"
    Stop:               "AGENT_STOP"
    SessionEnd:         "SESSION_END"

05 · FAQ

常见问题

MindPulse 和 Claude Code 自带的终端通知有什么区别？
终端通知需要你盯着终端窗口。MindPulse 把你的任务栏图标变成呼吸灯 — 你在浏览器、IDE、任何窗口都能用余光感知状态变化。不用切回去看。
是否免费？
完全免费，MIT 开源协议。无付费版、无遥测、无数据收集。
会占用多少系统资源？
运行时内存占用约 3–5 MB，CPU 占用在 60fps 渲染下约 0.1%。只在收到 HTTP 请求时才触发状态更新。
支持哪些 AI 工具？
内置 Claude Code 和 Codex 的事件映射配置。任何能发送 HTTP POST 的 AI 工具都可以通过添加对应的 YAML 配置文件来支持。
为什么是 127.0.0.1:9876？
仅在本地回环地址监听，不暴露到网络。端口 9876 可通过源码中的常量修改。
非 Windows 系统能用吗？
核心逻辑（状态机、动画引擎）完全跨平台。非 Windows 系统通过 TrayStub 降级为 stdout 日志输出。欢迎贡献 macOS / Linux 托盘实现。
